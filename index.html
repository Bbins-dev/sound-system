게임 사운드 시스템

곡을 만드는 데서 그치지 않고, 그 소리가 게임 상황에 반응해 ‘살아 움직이게’ 하는 시스템 자체를 직접 설계·구현했습니다.

BBSoundEngine

직접 만든 인터랙티브 사운드 미들웨어 · 실제 라이브 적용
사운드 디자이너인 제가 사운드 미들웨어를 직접 만들어, 회사 환경에 핏하게 설계해 끊김 없는 인터랙티브 음악 연출을 가능하게 했습니다. 미들웨어 부재로 컸던 개발자 커뮤니케이션 비용·업무 병목을 AI(Claude Code)를 적극 활용해 직접 풀어냈고, 개발본부 승인 → 라이브 서비스 적용 → 현재 신규 슬롯으로 확대 중입니다.
핵심 — 실시간 워크플로
개발자 빌드를 기다릴 필요가 없습니다. 사운드 담당자가 에디터에서 사운드 리소스와 재생 로직을 직접 교체·적용하면 구동 중인 게임에 실시간으로 반영되고, 인게임 상태(BGM·프리셋·더킹·LUFS/RMS)를 실시간 모니터링합니다. 협업 왕복과 빌드 대기 비용이 사라집니다.
  • 인터랙티브 BGM 레이어 크로스페이드 전환 — 같은 곡을 여러 레이어로 쌓아 두고 게임 상태에 따라 크로스페이드. 템포를 유지한 채 끊김 없이 고조·진정.
  • 인텐시티 단계 자동 전환 — 액션이 생기면 인텐시티 상승, 무동작이 지속되면 자동 디-에스컬레이션.
  • 자동 더킹 — 효과음 재생 시 BGM 게인을 자동으로 낮췄다 복구. 다중 더킹은 가장 강한 것 우선.
  • 이벤트·조건 기반 트리거 — “상황 → 사운드” 매핑을 조건 분기·시퀀스·랜덤 변주까지 지정.
  • 인하우스 경량 연동 — 외부 미들웨어(Wwise·FMOD) 도입 여건이 안 되는 환경에서, 기존 게임·워크플로에 그대로 임베드.
작동 시연 영상

Stick It!

혼자 만들어 출시한 Unity 모바일 게임 · 사운드 시스템 전부 직접 구축
Unity로 게임을 직접 기획·제작해 실제 출시했습니다. 그 게임의 사운드 시스템 전체(BGM 작곡·효과음 제작·재생 구조)를 혼자 구축했고, 아래는 그 실제 구현 사항입니다.
  • 속도 반응형 하이컷 + 소재별 랜덤마이즈 — 충돌 세기(속도)에 따라 하이컷(로우패스)이 달라짐(약하게=고역 감쇠, 세게=풀레인지). 피치·게인은 소재별 랜덤마이즈, 일부 소재는 여러 음원까지 랜덤마이즈해 매 타격이 다르게 들림.
  • 심리스 루프 연결 — 게이지 상승음을 샘플 정확도로 루프에 갭 없이 연결.
  • 보이스 분리 믹스 — 용도별 소스 분리로 보이스 스태킹·클리핑 방지.
  • 상태 기반 사운드 트리거 — 메뉴·조준·발사·성공·UI 등 게임 상태별 사운드 매핑.
  • 런타임 디테일 — 음소거 상태 영속화 / 앱 인터럽션 복귀 / 리플레이 시 이벤트 사운드 재발화 동기화(중복 방지).
실제 구현 화면
플레이·사운드 영상
두 프로젝트 모두 한 가지를 증명합니다 — 저는 곡을 만드는 데서 그치지 않고, 그 소리가 상황에 반응해 ‘살아 움직이게’ 하는 구조까지 직접 만듭니다. BBSoundEngine으로는 게임에서 감명받은 ‘템포를 유지하며 이어지는 인터랙티브 음악’을 실무에 구현했고, Stick It!에서는 충돌·게임 상태에 반응하는 사운드 시스템 전체를 직접 설계했습니다.
